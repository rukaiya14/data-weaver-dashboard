This file documents the AI-assisted prompts used with Kiro
to design, build, and refine the Data Weaver dashboard that
combines weather data with food delivery order trends.

---

## Prompt 1: Project Structure Creation

Create a frontend dashboard project with a clean folder structure
including HTML, CSS, JavaScript, a data folder for CSV files,
and a .kiro directory for prompts and context.

---

## Prompt 2: Dashboard UI Layout

Generate a clean and responsive HTML layout for a data analytics dashboard.
The dashboard should include:
- A header with title and subtitle
- Cards for weather information
- Chart sections for food orders and temperature trends
Use a modern, minimal design.

---

## Prompt 3: Weather API Integration

Generate JavaScript code to fetch real-time weather data
from the OpenWeatherMap API.
Display city name, temperature, humidity, and weather conditions
inside the dashboard.

---

## Prompt 4: CSV Data Loading

Generate JavaScript logic to load and parse a local CSV file
containing food order data with dates and order counts.
Prepare the data for visualization using Chart.js.

---

## Prompt 5: Data Mashup Logic

Combine weather data and food order data logically.
Identify whether weather conditions influence food ordering behavior
and suggest meaningful insights for visualization.

---

## Prompt 6: Chart Visualization

Generate Chart.js code to visualize:
- Food orders over time
- Temperature trends over time
Ensure charts are responsive and clearly labeled.

---

## Prompt 7: UI & UX Refinement

Refactor the dashboard UI to be production-ready:
- Improve spacing and alignment
- Use card-based layout with shadows and rounded corners
- Improve typography and color consistency
Do not change existing data logic.

---

## Prompt 8: Weather Impact Analysis Text

Generate a short, human-readable explanation for the
Weather Impact Analysis section based on observed trends.
Use simple rule-based logic instead of complex analytics.

---

## Prompt 9: Final Optimization

Optimize and clean the codebase for readability and maintainability.
Add comments where necessary and ensure best frontend practices.
